npm(node包管理工具)的使用

  • 第三方node模块（包）的管理工具，可以使用该工具下载安装第三方模块（包），当然也可以创建和上传发布自己的模块（包）

    技术框架介绍

  • node.js 基础核心开发语言
  • Express 一个简洁而灵活的node.js Web应用框架，提供一系列强大的特性帮助我们创建各种web应用
  • Mongodb

    第三方模块& 中间件

  1. bodyParser:解析post请求数据
  2. cookies:读/写cookie
  3. swig:模板解析引擎
  4. mongoose:操作mongodb数据
    5.markdown:markdown语法解析生成模块

    项目初始化

  • 在webstorm 左下角点击terminal
  • 输入npm init会生成一个package.json

    依赖模块安装

  • 安装express npm install –save express
  • 安装body-parser npm install –save body-parser
  • 安装cookies npm install –save cook# #ies
  • 安装markdown npm install –save markdown
  • 安装mongoose npm install –save mongoose
  • 安装swig npm install –save swig
  • 会在package.json文件下的

    项目结构分析

  • db 数据库存储目录
  • models 数据库模型文件目录
  • node_modules node第三方模块目录
  • public 公共文件目录（css,js,image…）
  • routers 路由文件目录
  • schemas 数据库结构文件目录（schema）’
  • views 模板视图文件目录
  • app.js 应用启动文件
  • package.json

    创建应用 监听端口

  • var express = require(‘express’)； 加载express模块
  • var app = express(); 创建app应用 等同于node.js中Http.createServer();
  • app.listen(8081);监听http请求

    用户的访问

  • 用户通过url访问web应用，如：http://localhost:8081/
  • web后端根据用户访问的url处理不同的业务逻辑

    处理请求输出

  • 通过app.get()或app.post()等方法可以把一个url路径和一个或n个函数进行绑定
  • app.get(‘/‘,function(req,res,next) {
  • req:request对象-保存客户端请求相关的一些数据-http.reqest
  • res:response对象-服务端输出的对象，提供了一些服务器端输出的相关方法-http.response，常用方法res.write(‘字符串或者标签’)，res.render(‘html文件’)，res.json(‘返回给前台一个对象’)
  • next:方法，用于执行下一个和路径匹配的函数
  • })

    内容输出

    通过res.send(string)发送内容至客户端

    使用模板

    模板的使用

    后端逻辑和页面表现分离 - 亲后端分离

    模板配置

  • var swig= require(‘swig’);加载模板处理模块
  • app.engine(‘html’,swig.renderFile);定义模板引擎，使用swig.renderFile方法解析后缀为html的文件，第一个单数末班引擎的名称，同时也是模板引擎的后缀。
  • app.set(‘views’,’./views); 设置模板文件存放目录，第一个参数必须是views,第二个参数是目录
  • app.set(‘view engine’,’html’); 注册所使用的模板引擎，第一个单数必须是’view engine’,第二个参数必须和app.engine(‘html’,swig.renderFile)方法的第一个参数一致。
  • swig.setDefaults({catche:false});

    res.render

    res.render(‘index’);读取views目录下的指定文件，解析并返回给客户端，第一个参数，表示模板的文件，相对于views目录第二个参数：传递给模板使用的数据

    静态资源文件处理（托管）

    app.use(‘/public’,express.static(dirname + ‘/public’));在public目录下划分并存放好相关的静态资源文件，当用户访问的url以/public开始，那么直接返回对应的dirname+’/public’下的文件
